# Дизайн-спецификация: Сравнение файлов

Дата: 2026-06-12

## Цель

Создать локальное Windows-приложение на Python для сравнения двух файлов любого типа. Пользователь выбирает исходный файл и измененную версию, запускает сравнение и видит, что было изменено, на что изменено, а также место изменения: строку, страницу, ячейку, координату или байтовое смещение.

Приложение работает в строгом локальном режиме: документы не отправляются в сеть, временные данные удаляются после закрытия или завершения сравнения, в логах не сохраняется текст документов.

## Название и визуальный стиль

Название продукта: **Сравнение файлов**.

Верхняя панель содержит компактный знак **СФ**, название **Сравнение файлов** и небольшую метку **2026**. В нижнем правом углу отображается приглушенная подпись:

**Разработал: Абдрахманов Амаль Даулетович**

Подпись должна выглядеть как аккуратная авторская метка, не конкурировать с рабочими элементами интерфейса и не отвлекать пользователя от результата сравнения.

Визуальный стиль: современный desktop-интерфейс 2026 года для Windows. Основной тон спокойный, рабочий, с четкой типографикой, плотной компоновкой и акцентами для изменений. Интерфейс не должен выглядеть как устаревшая админ-панель или декоративная посадочная страница.

## Основной интерфейс

Утвержденный вариант: документ-центричный экран.

Основные области:

1. Верхняя системная панель с названием, действиями открытия файлов, запуском сравнения, экспортом отчета и индикатором строгого локального режима.
2. Два синхронных просмотрщика: слева "Было", справа "Стало".
3. Правая панель со сводкой и списком изменений.
4. Нижняя строка состояния с техническими параметрами, локальным режимом и авторской подписью.

Основной пользовательский сценарий:

1. Пользователь выбирает два файла или перетаскивает их в окно.
2. Приложение определяет типы файлов и выбирает подходящий анализатор.
3. Пользователь запускает сравнение.
4. Приложение показывает список изменений.
5. Клик по изменению синхронно прокручивает оба просмотрщика к нужной строке, странице, ячейке или байтовому смещению.
6. Пользователь может отфильтровать изменения и экспортировать отчет.

## Режимы просмотра

В первой версии нужны режимы:

- **Текст**: построчный и пословный diff для текстовых документов, конфигов и извлеченного текста.
- **Страницы**: визуальный просмотр PDF, DOCX-рендера, изображений и OCR-страниц.
- **OCR**: просмотр распознанного текста с уровнем уверенности.
- **Hex**: fallback для неизвестных или бинарных файлов.

Интерфейс должен позволять переключаться между режимами без повторного выбора файлов.

## Поддерживаемые форматы

Приложение проектируется как система с расширяемыми анализаторами.

### Текст и config-файлы

Форматы:

- `.txt`, `.md`, `.log`, `.csv`, `.tsv`
- `.json`, `.yaml`, `.yml`, `.toml`
- `.ini`, `.cfg`, `.conf`, `.env`, `.properties`
- `.xml`, `.html`, `.css`, `.js`, `.py`, `.sql`
- `.gitignore`, `Dockerfile`, nginx/apache-подобные конфиги

Поведение:

- определение кодировки;
- нормализация переводов строк;
- построчный diff;
- пословный diff внутри измененных строк;
- структурный diff для известных config-форматов;
- вывод ключа или пути значения, например `database.host: old -> new`.

### Word-документы

Форматы:

- `.docx`

Поведение:

- извлечение абзацев, таблиц и базовой структуры;
- сравнение текста по абзацам и строкам;
- указание номера абзаца, таблицы или строки;
- сохранение связи результата с исходной частью документа.

Формат `.doc` считается устаревшим и поддерживается через внешний конвертер или fallback-режим, если надежный локальный парсер недоступен.

### PDF

Форматы:

- `.pdf`

Поведение:

- если в PDF есть текстовый слой, сначала используется он;
- если текстового слоя нет или он слабый, запускается локальный OCR;
- страницы рендерятся локально;
- изменения связываются со страницей, строкой OCR и координатами области.

### Excel

Форматы:

- `.xlsx`, `.xlsm`

Поведение:

- сравнение книг, листов, строк, столбцов и ячеек;
- отображение старого и нового значения;
- указание листа и адреса ячейки, например `Sheet1!B7`;
- сравнение формул как текста формул и видимых значений, если они доступны.

Формат `.xls` поддерживается через отдельный анализатор или fallback-конвертацию, если это потребуется в рабочей среде.

### Изображения и сканы

Форматы:

- `.png`, `.jpg`, `.jpeg`, `.bmp`, `.tiff`, `.webp`

Поведение:

- визуальное сравнение изображений;
- выделение отличающихся областей;
- OCR для распознавания текста;
- привязка изменений к координатам и распознанным строкам;
- отображение уровня уверенности OCR.

### Неизвестные и бинарные файлы

Поведение:

- сравнение размера и хэша;
- бинарное сравнение блоков;
- Hex-просмотр отличий;
- указание байтового смещения и диапазона изменения;
- понятное сообщение, что строковое представление недоступно.

## Архитектура

Приложение делится на слои:

1. **UI слой**: PySide6/Qt desktop-интерфейс.
2. **Application слой**: сценарии выбора файлов, запуска сравнения, отмены, прогресса, экспорта.
3. **Core diff слой**: единая модель результата и алгоритмы сопоставления.
4. **Parser/Analyzer слой**: анализаторы конкретных типов файлов.
5. **OCR слой**: локальный OCR через Tesseract.
6. **Security слой**: строгий локальный режим, временные файлы, логи, очистка.
7. **Reporting слой**: экспорт результатов в HTML/PDF/JSON по мере необходимости.

### Единая модель результата

Каждый анализатор возвращает список изменений в общем формате:

- `change_id`
- `change_type`: added, removed, modified, moved, metadata, binary
- `severity`: info, normal, warning, risk
- `source_location`
- `target_location`
- `before`
- `after`
- `format`
- `confidence`
- `notes`

Локация зависит от типа файла:

- текст: файл, строка, колонка;
- PDF/OCR: страница, строка OCR, координаты;
- Excel: лист, ячейка;
- изображение: координаты области;
- бинарный файл: байтовое смещение и диапазон.

## Основные библиотеки

Базовый стек:

- Python 3.12+
- PySide6/Qt для Windows GUI
- Tesseract OCR с языками `rus` и `eng`
- PyMuPDF или pypdfium2 для PDF-рендера и извлечения страниц
- openpyxl для `.xlsx` и `.xlsm`
- python-docx для `.docx`
- Pillow и OpenCV/scikit-image для изображений
- difflib или специализированный diff-алгоритм для текстового сравнения
- PyInstaller или Nuitka для сборки `.exe`

## Безопасность и конфиденциальность

Приложение работает по принципу privacy-first:

- никакой отправки документов в сеть;
- никакой облачной OCR-обработки;
- временные файлы создаются только в контролируемой временной директории приложения;
- временные файлы удаляются после завершения сравнения или закрытия приложения;
- логи не содержат текст документов, OCR-текст, имена конфиденциальных сущностей или фрагменты файлов;
- в логах допустимы только технические события: тип анализатора, длительность операции, код ошибки, размер файла, но не содержимое;
- при ошибке пользователь видит понятное сообщение без раскрытия содержимого документа.

## Ошибки и ограничения

Приложение должно явно объяснять ограничения:

- если файл поврежден;
- если формат не поддержан специализированным анализатором;
- если OCR дал низкую уверенность;
- если документ защищен паролем;
- если файл слишком большой для выбранного режима;
- если невозможно показать строку и доступен только бинарный diff.

В таких случаях приложение не должно падать. Оно должно перейти в ближайший безопасный fallback-режим.

## Производительность

Сравнение выполняется в фоновом потоке, чтобы UI не зависал.

Требования:

- прогресс по этапам: чтение, парсинг, OCR, сравнение, построение отчета;
- возможность отмены;
- ограничение памяти для больших файлов;
- ленивый рендер страниц и изображений;
- обработка OCR постранично.

## Тестирование

Нужны тесты на:

- текстовый diff;
- config structural diff;
- `.docx` extraction;
- `.pdf` text-layer extraction;
- OCR fallback для сканированных PDF;
- `.xlsx` cell diff;
- image diff;
- binary fallback;
- strict mode: отсутствие содержимого документов в логах;
- корректную очистку временных файлов;
- корректную модель результата для всех анализаторов.

Также нужен набор тестовых файлов с искусственными изменениями. Тестовые документы не должны содержать реальные персональные или конфиденциальные данные.

## Сборка и поставка

Первая целевая платформа: Windows.

Поставка:

- `.exe` или инсталлятор;
- bundled runtime Python;
- bundled Tesseract или проверка установленного Tesseract при первом запуске;
- языковые пакеты OCR `rus` и `eng`;
- конфигурация strict mode включена по умолчанию.
